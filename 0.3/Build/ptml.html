ADD
Add two numeric values
Syntax:
ADD a, b
Calculates a = a + b
  • a = Variable identifier
  • b = Variable identifier or number literal
ADD a, b, c
Calculates a = b + c
  • a = Variable identifier
  • b = Variable identifier or number literal
  • c = Variable identifier or number literal
ASC
Get ASCII value of character
Syntax:
ASC var, character
Get ASCII value of character, then store it in the given variable.

If a literal string is given instead of a literal character, only the first character of the string is checked. If a variable is given as the second argument, the value of the variable is first converted to a string and then the first character is checked.
  • var = Variable identifier
  • character = The character to get the ASCII value from
AUTOREF
Enable or disable automatic screen refresh
Syntax:
AUTOREF state
Enable or disable the automatic update of the screen. When enabled, the machine automatically copies the entire tile buffer to the screen on every frame, which can significantly reduce performance. To speed up performance, disable the automatic refresh, and execute the REFRESH command when needed to manually update the screen.
  • state = 0: disabled; 1: enabled
BEEP

Syntax:

  • freq = The sound frequency
  • length = The beep duration
BEEP
Emit the standard PTM beeping sound (frequency: 1450, length: 50)

CHR
Set pixel pattern for character at charset index
Syntax:
CHR charset_index, 64_bit_pattern
Set pixel pattern for character at charset index
  • charset_index = Charset index to alter
  • 64_bit_pattern = Binary string of length 64, where "0" means a background pixel, and "1" means a foreground pixel in the pixel pattern. The string gets split into 8 substrings, each one of length 8, for each row of the pattern.
CHR charset_index, pattern_row, 8_bit_pattern
Set pixel pattern for given pattern row of character at charset index
  • charset_index = Charset index to alter
  • pattern_row = Row index to alter
  • 8_bit_pattern = Binary string of length 8, where "0" means a background pixel, and "1" means a foreground pixel in the pixel pattern of the given row.

INPUT
Wait for text input from user
Syntax:
INPUT var
Causes program to wait for user to write a line of text. When user presses the ENTER key, the text is stored in the given variable and program resumes execution. If user presses the ESCAPE key, an empty string is stored in the variable instead. By default, the user will not be able to type more than 255 characters.
  • var = Variable id
INPUT var, default_value
The same as the previous syntax, but if user presses the ESCAPE key, the default_value is stored in the variable instead of an empty string.
  • var = Variable id
  • default_value = The value to assign to variable if ESCAPE key is pressed
INPUT var, default_value, max_length
The same as the previous syntax, but limits input to the given maximum length. The user will not be able to type more characters than the specified limit.
  • var = Variable id
  • default_value = The value to assign to variable if ESCAPE key is pressed
  • max_length = The maximum length of the text

MODE
Switch color mode
Syntax:
MODE mode
Switch the color mode.

Monochrome mode is the default, where every tile on the screen has the same foreground and background color, and the COLOR, INK and PAPER commands change the color of all tiles on the screen at once.

Multicolor mode prevents the COLOR, INK and PAPER commands to change existing tiles, and instead only sets the color of subsequent tiles.
  • mode = 0: monochrome; 1 = multicolor

PLAY
Play a sequence of sound tones (MML)
Syntax:
PLAY mml
Play a sequence of MML commands and sound tones from a string. Each MML command and tone in the string must be separated by a space character.
  • mml = String of MML commands and sound tones
PLAY mml, loop
The same as the first syntax, optionally looping when finished.
  • mml = String of MML commands and sound tones
  • loop = 0: play only once; 1: loop indefinitely
MML commands:
  • L = Note length. All subsequent notes will be played with this newly specified length.
  • P = Pause. Pauses MML execution for the specified interval.
MML sound tones:
Each tone must be followed by an octave, from 0 to 8.
  • C = Do
  • C# = Do (sharp)
  • D = Re
  • D# = Re (sharp)
  • E = Mi
  • F = Fa
  • F# = Fa (sharp)
  • G = Sol
  • G# = Sol (sharp)
  • A = La
  • A# = La (sharp)
  • B = Si

PRINT

Syntax:

Print text on the screen at current cursor position. The cursor automatically moves forward to the next column after the last printed character.
  • string = The text to print
PRINTF
Print formatted text on screen at current cursor position
Syntax:
PRINTF format, value
Print formatted text on screen at current cursor position. The format string is similar to the C language. However, the only supported format specifiers are %i (integer) and %x (hexadecimal), with optional padding zeroes (e.g. %05i). Only a single specifier is interpreted in the string, because PTML does not support variable length arguments.
  • format = The format string (similar to the C language)
  • value = The value (or variable) to be formatted in the string
PRINTL

Syntax:

Print text on the screen at current cursor position, then move cursor to next line, at the same column where the previous PRINTL command started in the previous line.
  • string = The text to print
PRINTL string, keep_column
Print text on the screen at current cursor position, then move cursor to next line. If keep_column = 1, the next line begins at the same column where the previous PRINTL command started in the previous line. If keep_column = 0, the next line starts at column 0.
  • string = The text to print
  • keep_column = 0: no; 1: yes
PRINTL

SPLIT
Split string into array
Syntax:
SPLIT array_name, string, delimiter
Split string at the given delimiter, then each part gets assigned to each element of the specified array. If the array is not long enough to contain all parts, the remaining parts are discarded.
  • array_name = Name of the array that will contain the string parts
  • string = The string to split
  • delimiter = The delimiter at which to split the string. It should be a literal character or a number. If a string (or variable that contains a string) is given, only the first character of the string is taken as the delimiter.
SPRITE
Create or update sprite
Syntax:
SPRITE id, x, y, transparency, visible
Create sprite with tile equal to tile register, position, transparency and visibility

Sprites are not part of the screen tile buffer, therefore their coordinates are pixel-based, rather than tile-based

"Transparent" sprites have only a single color (their foreground pixels), while their background pixels are completely hidden, allowing the colors of underlying tiles to be seen through.
  • id = Sprite id
  • x = X-coordinate on the screen
  • y = Y-coordinate on the screen
  • transparency = 0: opaque; 1: transparent
  • visible = 0: not visible; 1: visible
SPRITE id
Set sprite tile equal to tile register, without changing any other property
  • id = Sprite id
SPRX
Get sprite X position
Syntax:
SPRX id, var
Get sprite X position and store it in the variable
  • id = Sprite id
  • var = Variable identifier
SPRY
Get sprite Y position
Syntax:
SPRY id, var
Get sprite Y position and store it in the variable
  • id = Sprite id
  • var = Variable identifier
SQRT
Calculate square root of a numeric value
Syntax:
SQRT var, value
Calculate var = √value (square root)
  • var = Variable identifier
  • value = The base value
SUB
Subtract two numeric values
SUB a, b
Calculates a = a - b
  • a = Variable identifier
  • b = Variable identifier or number literal
SUB a, b, c
Calculates a = b - c
  • a = Variable identifier
  • b = Variable identifier or number literal
  • c = Variable identifier or number literal
SUBS
Get substring
Syntax:
SUBS var, source_string, first_index, last_index
Extract a substring from the given string, that starts at the first_index and ends at the last_index. The source string is not modified.
  • var = Variable identifier that will receive the substring
  • source_string = Source string
  • first_index = The first index of the substring
  • last_index = The last index of the substring
SUBS var, source_string, first_index
The same as the previous syntax, but the last_index is implied to be the index of the last character of the source string.
  • var = Variable identifier that will receive the substring
  • source_string = Source string
  • first_index = The first index of the substring
SWAP
Swap value between two variables
Syntax:
SWAP var_1, var_2
Swap value between two variables
  • var_1 = First variable
  • var_2 = Second variable
TILE
Set tile register
Syntax:
TILE ch, fgc, bgc
Append a new character with given colors to the end of the tile register
  • ch = Charset index
  • fgc = Palette index for the foreground color
  • bgc = Palette index for the background color
TILE
Remove all characters from the tile register
TILESPR
Copy sprite tile to tile register
Syntax:
TILESPR id
Copy sprite tile to tile register
  • id = Sprite id
TITLE
Set the program title
Syntax:
TITLE title
Set the program title. The title appears as the window caption of the host operating system
  • title = The string to be used as program title
UCASE
Convert string to uppercase
Syntax:
UCASE var, string
Convert to uppercase all characters of string, then store it in the given variable. The original string is not modified.
  • var = Variable identifier
  • string = The string to convert
UCASE var
Convert to uppercase all characters of string stored in the given variable.
  • var = Variable identifier which has the string to be converted
WRITE
Write to data record file
Syntax:
WRITE value
Write data to currently open record file, then moves the write pointer to the next record.

To write data, the file must be opened with the OPEN command in mode "W" (write). Data record files are buffered, meaning the physical file on disk is not modified until the CLOSE command is executed, which then saves the buffer to the file.
  • value = Value to write
